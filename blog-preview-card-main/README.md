## Di mana menemukan segalanya

Tugas Anda adalah membangun proyek sesuai dengan desain di dalam folder `/design`. Anda akan menemukan versi mobile dan desktop dari desain.

Desain dalam format JPG statis. Menggunakan JPG berarti Anda perlu menggunakan penilaian terbaik untuk gaya seperti `font-size`, `padding` dan `margin`.

Jika Anda ingin file desain (kami menyediakan versi Sketch & Figma) untuk memeriksa desain secara lebih detail, Anda dapat [berlangganan sebagai anggota PRO](https://www.frontendmentor.io/pro).

Semua aset yang diperlukan untuk proyek ini ada di folder `/assets`. Gambar sudah diekspor untuk ukuran layar yang benar dan dioptimalkan.

Kami juga menyertakan file font variabel dan statis untuk font yang diperlukan untuk proyek ini. Anda dapat memilih untuk menautkan ke Google Fonts atau menggunakan file font lokal untuk menghosting font sendiri. Perhatikan bahwa kami telah menghapus file font statis untuk bobot font yang tidak diperlukan untuk proyek ini.

Ada juga file `style-guide.md` yang berisi informasi yang Anda butuhkan, seperti palet warna dan font.

## Membangun proyek Anda

Silakan gunakan alur kerja apa pun yang Anda rasa nyaman. Di bawah ini adalah proses yang disarankan, tetapi jangan merasa perlu mengikuti langkah-langkah ini:

1. Inisialisasi proyek Anda sebagai repositori publik di [GitHub](https://github.com/). Membuat repo akan memudahkan berbagi kode Anda dengan komunitas jika Anda membutuhkan bantuan. Jika Anda tidak yakin cara melakukannya, [baca sumber daya Try Git ini](https://try.github.io/).
2. Konfigurasikan repositori Anda untuk menerbitkan kode ke alamat web. Ini juga akan berguna jika Anda membutuhkan bantuan selama tantangan karena Anda dapat berbagi URL proyek Anda dengan URL repo. Ada beberapa cara untuk melakukannya, dan kami menyediakan beberapa rekomendasi di bawah.
3. Lihat desain untuk mulai merencanakan bagaimana Anda akan menangani proyek. Langkah ini penting untuk membantu Anda berpikir ke depan untuk kelas CSS untuk membuat gaya yang dapat digunakan kembali.
4. Sebelum menambahkan gaya apa pun, struktur konten Anda dengan HTML. Menulis HTML Anda terlebih dahulu dapat membantu fokus perhatian Anda pada pembuatan konten yang terstruktur dengan baik.
5. Tulis gaya dasar untuk proyek Anda, termasuk gaya konten umum, seperti `font-family` dan `font-size`.
6. Mulai menambahkan gaya dari atas halaman dan bekerja ke bawah. Hanya lanjut ke bagian berikutnya setelah Anda puas telah menyelesaikan area yang sedang Anda kerjakan.

## Buat `README.md` kustom

Kami sangat merekomendasikan menimpa `README.md` ini dengan yang kustom. Kami telah menyediakan template di dalam file [`README-template.md`](./README-template.md) di kode starter ini.

Template memberikan panduan tentang apa yang harus ditambahkan. `README` kustom akan membantu Anda menjelaskan proyek Anda dan merefleksikan pembelajaran Anda. Silakan edit template kami sebanyak yang Anda suka.

Setelah Anda menambahkan informasi ke template, hapus file ini dan ganti nama file `README-template.md` menjadi `README.md`. Itu akan membuatnya muncul sebagai file README repositori Anda.
